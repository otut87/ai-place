# AI플레이스 페이지 구조 리뷰

> 리뷰 대상 2종:
> 1. `aiplace.kr/cheonan/webagency` — 카테고리 리스팅 (웹에이전시)
> 2. `aiplace.kr/guide/cheonan/dermatology` — 가이드 페이지 (피부과 선택 가이드)
>
> 리뷰 일자: 2026-04-15

---

## 페이지 유형 정리

두 페이지는 역할이 명확히 다름:

- **카테고리 리스팅** (`/cheonan/webagency`) — 특정 지역+업종의 업체 목록 페이지. 현재 디두 1곳 등록. ItemList + ProfessionalService 스키마 적용.
- **가이드 페이지** (`/guide/cheonan/dermatology`) — 피부과 선택을 돕는 에디토리얼 콘텐츠. Article + Person(저자) 스키마 적용. 블로그 발행물에 해당.

---

## 현재 잘 되어 있는 것

### 공통

- **Next.js SSR 렌더링** — AI 봇이 JavaScript 없이도 콘텐츠를 읽을 수 있음
- **페이지별 고유 H1** — 리스팅: `"천안 웹에이전시 추천 — 2026년 업데이트"`, 가이드: `"천안 피부과 선택 가이드 — 2026년 업데이트"`. 사이트 공통 H1 대신 페이지 목적에 맞는 고유 제목 적용됨
- **BreadcrumbList JSON-LD** — 리스팅은 홈→천안 웹에이전시 (2단계), 가이드는 홈→천안 피부과→선택 가이드 (3단계). 사이트 계층 구조를 AI에게 정확히 전달
- **카테고리 특화 FAQ + FAQPage 스키마** — 리스팅에 웹에이전시 FAQ 3개 (비용, 기간, 선택 기준), 가이드에 피부과 FAQ 6개 (선택법, 비용, 주말진료, 보험, 야간진료, 준비물). 사이트 소개 FAQ와 분리됨
- **메타 태그 풀세트** — title, description, og:title/description/url, twitter:card/title/description, canonical
- **robots: index, follow** + Google/Naver 사이트 인증 완료
- **최종 업데이트 날짜 표시** — 리스팅 2026-04-16, 가이드 2026-04-14
- **H1 아래 요약문 배치** — 페이지 목적을 설명하는 서브헤드라인 존재

### 가이드 페이지 전용

- **Article 스키마** — headline, description, dateModified(2026-04-14), author(이지수, AI Place 큐레이터), publisher(AI 플레이스) 완비
- **Person 스키마** — 저자 정보(이름, 직함, URL)로 E-E-A-T 신뢰도 확보
- **에디토리얼 본문 8개 섹션** — 선택 시 고려할 점, 증상별 추천, 비용 안내, 진료 시간 비교, 첫 방문 준비사항, 온라인 예약, 시술별 비교, 참고 출처. 리스팅과 완전히 차별화된 콘텐츠
- **구체적 데이터 포함** — 업체별 정확한 진료 시간, 시술별 가격대, 야간진료 시간까지 명시
- **의료 면책문** — "본 페이지는 정보 제공 목적이며, 실제 비용은 상담 후 확정됩니다" — YMYL 콘텐츠에 적절한 고지

### 카테고리 리스팅 전용

- **ItemList 스키마** — ProfessionalService 타입 + AggregateRating(별점 5, 리뷰 2건) 포함
- **업종 특화 FAQ** — 홈페이지 제작 비용(100-300만원), 제작 기간(1주~3개월), 선택 기준 등 실용적 답변

> **이전 리뷰 정정:** v1에서 "H1이 모든 페이지 동일", "BreadcrumbList 없음", "FAQ가 AI Place 소개만"이라고 했으나 모두 오류. 실제로는 페이지별 고유 H1, BreadcrumbList, 카테고리 특화 FAQ가 이미 적용되어 있음.

## 수정 필요 사항 (우선순위순)

### 1. 메타 디스크립션 및 본문 템플릿 오류 — 즉시 수정

**문제:** 웹에이전시 페이지에 의료 카테고리 전용 문구가 잔존

발생 위치 2곳:
- meta description: `"천안시에 위치한 웹에이전시 목록. 진료 과목, 위치, 리뷰 기반 정리."`
- H1 아래 요약문: `"천안시에 위치한 웹에이전시 1곳을 진료 과목, 위치, 이용 후기 기준으로 정리했습니다."`

"진료 과목"은 의료업종에만 해당하는 표현. 웹에이전시 페이지에서는 의미가 맞지 않음.

**수정 방향:**
- 업종별 메타 디스크립션 템플릿 분기 처리
- 웹에이전시: `"천안시에 위치한 웹에이전시 목록. 서비스 분야, 포트폴리오, 리뷰 기반 정리."`
- 본문 요약: `"천안시에 위치한 웹에이전시 1곳을 서비스 분야, 위치, 이용 후기 기준으로 정리했습니다."`
- 코드에서 `category.type === 'medical'` 분기를 확인하고, 비의료 업종에 "진료 과목"이 노출되지 않도록 수정

---

### 2. 크로스 카테고리 업체 카드 노출 — 구조적 문제

**문제:** 웹에이전시 리스팅 페이지에 피부과 업체 5곳(수피부과, 닥터에버스, 클린휴, 샤인빔, 얼라이브)의 카드가 함께 노출됨. HTML 소스에 피부과 업체 주소, 설명, 야간진료 태그까지 포함.

JSON-LD ItemList에도 별도의 피부과 ItemList 스키마가 삽입되어 있음:
```
"천안 피부과 추천 업체", "numberOfItems": 5
```

**위험:** AI 크롤러가 웹에이전시 페이지를 피부과 관련 페이지로 혼동할 수 있음. 해당 카테고리와 무관한 스키마가 같은 페이지에 있으면 토픽 일관성(topical relevance)이 훼손됨.

**수정 방향:**
- 웹에이전시 페이지에서 피부과 업체 카드 및 피부과 ItemList 스키마 제거
- "다른 카테고리 둘러보기"는 텍스트 링크로만 유지 (카드 목록 X)
- 또는 별도 섹션으로 분리하되 `"관련 카테고리"` 명시, JSON-LD에는 포함하지 않기

---

### 3. Direct Answer Block 강화 — GEO 핵심

**문제:** 요약문이 존재하지만 AI가 직접 인용하기엔 정보 밀도가 부족. 현재 형태:

리스팅: `"천안시에 위치한 웹에이전시 1곳을 진료 과목, 위치, 이용 후기 기준으로 정리했습니다."` — 업체명, 평점, 핵심 서비스가 없음

가이드: `"천안 지역 피부과 5곳의 전문 분야, 비용, 진료 시간을 종합 정리한 선택 가이드입니다."` — 어떤 업체인지 알 수 없음

**수정 방향:** H1 바로 아래에 AI가 그대로 뽑아갈 수 있는 자기완결형 요약 배치

```
리스팅 예시:
"2026년 4월 기준, 천안 지역 웹에이전시 '디두'는 웹사이트 제작과
디지털 마케팅을 전문으로 하며, 기본 홈페이지 제작 100-300만원,
쇼핑몰 300-800만원 수준입니다. (★5.0, 리뷰 2건)"

가이드 예시:
"천안에서 피부과를 선택할 때는 전문 분야, 접근성, 야간진료 여부를
기준으로 비교하세요. 수피부과의원(일반피부질환), 닥터에버스(리프팅·보톡스),
클린휴(기미·모공), 샤인빔(스킨부스터), 얼라이브(난치성여드름·탈모)가
천안 주요 피부과입니다."
```

**기술 구현:** `<p>` 태그에 `data-answer-block` 속성 부여, 40~80자 내외로 핵심 정보 압축

### 4. `<time>` 시맨틱 태그 미적용

**문제:** "최종 업데이트: 2026-04-16" 텍스트는 있으나 HTML `<time datetime="2026-04-16">` 태그로 마크업되어 있지 않음. AI 크롤러는 `<time>` 태그의 `datetime` 속성을 통해 콘텐츠 최신성을 기계적으로 판단함.

**수정 방향:**
```html
<time datetime="2026-04-16">최종 업데이트: 2026-04-16</time>
```
- Article 스키마의 `dateModified`와 `<time>` 태그의 날짜를 동기화
- 데이터 변경 시 자동 갱신되는 구조

---

### 5. AI Place 서비스 FAQ 분리

**문제:** 카테고리 특화 FAQ와 AI Place 서비스 소개 FAQ ("AI Place는 무엇인가요?", "업체 등록은 어떻게 하나요?" 등)가 같은 페이지에 혼재. 두 종류 모두 FAQPage 스키마에 들어가면, AI가 "천안 피부과" 검색 시 서비스 소개 질문까지 인용할 수 있음.

**수정 방향:**
- 서비스 FAQ는 footer 영역이나 `/about` 페이지로 이동
- 각 카테고리 페이지의 FAQPage 스키마에는 해당 업종 FAQ만 포함
- 서비스 FAQ는 별도 FAQPage 스키마를 `/about`에 적용

---

### 6. og:image 미설정

**문제:** 두 페이지 모두 og:image가 없음. SNS 공유 시 썸네일이 표시되지 않아 클릭률이 낮고, 이미지 검색 노출 기회를 놓침.

**수정 방향:**
- 카테고리별 자동 생성 OG 이미지 (예: 업종 아이콘 + 지역명 + 업체 수)
- 가이드 페이지는 에디토리얼 느낌의 커버 이미지
- Next.js의 `@vercel/og` 또는 사전 생성 방식으로 자동화

---

### 7. robots.txt AI 봇 명시 허용 + llms.txt 생성

**권장:** 현재 robots.txt 상태를 확인하고, AI 크롤러별 명시적 허용 추가

```
User-agent: GPTBot
Allow: /

User-agent: ChatGPT-User
Allow: /

User-agent: OAI-SearchBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: *
Allow: /
Sitemap: https://aiplace.kr/sitemap.xml
```

**llms.txt 생성:**

```
# AI Place
> AI가 추천하는 우리 동네 업체 디렉토리

AI Place는 한국 로컬 사업체 정보를 AI 검색 엔진이 읽을 수 있는 구조화된 형태로 제공합니다.
현재 천안 지역 피부과, 웹에이전시 등 8개 업체가 등록되어 있습니다.

## 주요 페이지
- [천안 피부과](https://aiplace.kr/cheonan/dermatology): 천안 피부과 5곳 비교
- [천안 웹에이전시](https://aiplace.kr/cheonan/webagency): 천안 웹에이전시 목록
- [피부과 선택 가이드](https://aiplace.kr/guide/cheonan/dermatology): 천안 피부과 선택 가이드
- [업체 등록](https://aiplace.kr/admin/register): 업체 등록 안내
```
