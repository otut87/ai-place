# AI Place 사업 아이템 최종 종합 리뷰

지금까지 논의한 모든 맥락을 종합해 최종 정리해드리겠습니다. 초기 플랫폼 검토부터 비즈니스 모델, 디자인 전략까지 하나의 그림으로 묶어보겠습니다.

---

## 1. 사업 개요 재정의

**AI Place (aiplace.kr)**
- **운영 주체**: 디두(dedo) — 천안 기반 웹디자인·마케팅 에이전시
- **핵심 가치 제안**: 홈페이지 없는 소상공인에게 **AI 검색(ChatGPT·Claude·Gemini) 노출을 위한 구조화된 업체 페이지**를 월 33,000원 구독으로 제공
- **본질**: 사용자 대면 디렉토리가 아닌 **"AI 크롤러를 위한 업체 정보 SaaS"**
- **포지셔닝**: 디두 본업과 2단 가격 구조로 역할 분담
  - 하이엔드 (디두): 홈페이지 제작 + AIO 컨설팅, 수백만 원
  - 로우엔드 (AI Place): 구독형 AI 노출 페이지, 월 3.3만 원
- **현재 상태**: 프로덕션 배포 후 1주 미만, 시연용 데모 단계

## 3. 핵심 강점 (왜 성공 가능성이 높은가)

### 전략적 강점
- **타이밍**: AI 검색이 전통 SEO를 대체하는 시기에 정확히 진입
- **2단 가격 구조**: 로우엔드(AI Place)가 하이엔드(디두)를 카니발하지 않고 상호 보완
- **업셀 퍼널**: 월 3.3만 원 고객 중 10%를 디두 본업으로 전환 가능
- **단면 SaaS 특성**: 일반 디렉토리와 달리 양면 시장의 Cold Start 문제 회피

### 제품 강점
- **AI 크롤러 친화적 기술 기반**: robots.txt에 10여 종 AI 봇 명시 허용, Schema.org JSON-LD(LocalBusiness·MedicalClinic·FAQ·ItemList·AggregateRating) 정확한 적용
- **콘텐츠 구조**: 업체 상세 페이지에 주소·전화·서비스·가격대·리뷰·FAQ 5개·관련 콘텐츠까지 AI 인용에 최적화된 깊이
- **E-E-A-T 설계**: 가이드·비교 페이지에 대한피부과학회·건강보험심사평가원 등 권위 있는 출처 인용

### 가격 전략 강점
- **심리적 임계가격 적중**: 월 3.3만 원은 "일 1,100원"으로 환산되어 의사결정 마찰 최소화
- **증명 압박 완화**: 월 300만 원이면 ROI 증명 부담 크지만, 3.3만 원은 "해봐서 나오면 좋고" 범주
- **경쟁사 진입 장벽**: 저가격대는 규모 없이는 수익 안 남아 신규 경쟁자 진입 억제

## 4. 주요 리스크와 대응 전략

### 리스크 1: AI 크롤러 정책 변경 (최상위 리스크)
AI 플랫폼이 저품질 디렉토리를 차단하거나 인용 중단하면 서비스 가치가 즉시 0이 됨. 이미 Google은 "파라사이트 SEO" 페널티 시행 중.

**대응**: 업체별 고유 콘텐츠 최소 500~1,000자 확보, 템플릿 중복도 억제, 업체 직접 작성 정보 비중 확대

### 리스크 2: 증명 가능성의 본질적 한계
AI 검색 결과는 개인화·랜덤화되어 "내 업체가 나온다"를 객관적으로 증명하기 어려움.

**대응**: 월 1회 리포트 제공 — 주요 쿼리 조합으로 ChatGPT·Claude·Gemini 검색 스크린샷 + AI 봇 크롤링 로그 기반 방문 횟수 분석. 이 리포트가 락인의 핵심 무기

### 리스크 3: 의료 카테고리 법적 리스크
피부과 시술명·가격 표기는 의료법상 광고 규제 대상이며, 의료광고심의 대상 가능성.

**대응**: 의료 카테고리 전용 정책 수립, 가격은 "상담 문의"로 대체 또는 명시적 심의 필증 확보

### 리스크 4: 구글·네이버 리뷰 스크래핑 저작권
현재 리뷰 본문을 표시하는 방식은 저작권·이용약관 이슈 소지.

**대응**: 리뷰는 "출처 링크"로만 표시하고 본문 비스크래핑, 또는 Google Places API 공식 사용

### 리스크 5: 1인 운영의 번아웃
업체 300곳 넘으면 수동 운영 불가능. 업체당 월 10분 이하로 운영 시간 축소 필수.

**대응**: 업체 등록 → 페이지 생성 → 월 리포트 발송까지 **완전 자동화 파이프라인** 구축. 이게 없으면 스케일업 불가

### 리스크 6: AI 플랫폼의 직접 흡수
OpenAI·Google이 Business Listings 서비스를 직접 런칭하면 중개자 가치 소멸.

**대응**: 디두 본업 업셀 구조를 먼저 확립, AI Place를 독립 플랫폼이 아닌 디두 서비스 쇼케이스로 포지셔닝하면 흡수 리스크 완화

## 5. 현재 프로덕션 상태 진단

### 즉시 수정 필요 (신뢰도 이슈)
- **통계 수치 불일치**: 홈 "83개 업종" / "8곳" / 피부과 "4곳" vs 가이드 "5곳" vs 여드름 "3곳" → 데이터 소스 단일화 필수
- **템플릿 복사 버그**: 웹에이전시 페이지 하단에 "의료 결정은 전문의와 상담" disclaimer 잔존 → 카테고리별 분기 처리
- **업체 수 부족 카테고리**: 자동차정비·인테리어·웹에이전시·맛집 각 1곳만 등록 → "준비 중"으로 숨기거나 최소 3곳 확보 후 공개

### 런칭 전 보완 필요
- 가격 페이지 신설 (현재 가격 정보 부재)
- 업체 등록 플로우 완성 (입력 → 결제 → 페이지 생성까지)
- AI 검색 리포트 프로토타입 제작
- 템플릿 자동화 파이프라인 구축
- 의료 카테고리 컴플라이언스 점검
- 네비게이션·검색 기능 추가 검토

## 6. 실행 전략 (우선순위 기반)

### Phase 1: 런칭 전 정비 (2~4주)
신뢰도 이슈 전면 수정, 자동화 파이프라인 구축, 가격·등록·결제 플로우 완성, 피부과 1~2개 지역만 공개로 완성도 있는 데모 구성

### Phase 2: 베타 런칭 (1~2개월)
디두 기존 고객 10~20곳 무료 체험 등록, "100호 고객까지 월 19,800원" 얼리버드 특가, 후기·사례 확보, 지역 커뮤니티·SNS 콘텐츠 마케팅

### Phase 3: 정식 런칭 (3~6개월)
가격 정상화(얼리버드는 평생 동결), 추천 프로그램 도입, 업종 확장(치과·한의원·헬스장·미용실 순), 디두 본업 업셀 루프 가동

### Phase 4: 스케일업 (6~12개월)
업체 대시보드 MVP, AI 언급 트래킹 기능, 천안 외 지역 확장, 광고 집행 시작

## 7. 아키텍처 전략

### 1사이트 2트랙 구조
하나의 도메인(aiplace.kr)에서 역할이 다른 두 영역을 운영하는 방식이 최적:

**AI 전용 디렉토리 페이지** (/cheonan/*, /guide/*, /compare/*)
- 현재 구조 유지
- AI 크롤러 중심 설계
- 정보 밀도 우선
- 최소한의 시각적 요소

**랜딩 페이지** (/, /pricing, /register, /how-it-works)
- 사람 대상 세일즈
- Airbnb 스타일 따뜻한 톤
- 전환율 최적화
- 신뢰 요소·사회적 증거 강조

두 영역이 **동일 디자인 시스템**을 공유하되 컴포넌트 활용 밀도가 다른 구조. 본인의 디자인 시스템 문서가 이미 이 통합을 잘 지원하고 있음.

### 업체 페이지의 이중 역할
AI 크롤러용 페이지지만 실제 업체 사장님이 "내 페이지 확인" 용도로 방문하게 됨. 이 시나리오에서 랜딩 → 업체 페이지 → 다시 랜딩의 플로우가 자연스러워지므로 **동일 디자인 시스템이 필수**. 이 판단은 이미 옳음.

## 8. 디자인·개발 워크플로우

### 확정된 방식
- **디자인**: 본인이 Figma로 직접 작업
- **개발**: Claude Code로 구현
- **디자인 시스템**: Airbnb 기반 Place Green 단일 액센트 시스템 (이미 aiplace.kr 적용 중)
- **Figma 토큰 관리**: Tokens Studio 플러그인 + JSON 기반 (향후 Style Dictionary로 코드 연동 가능)

### 이 조합의 강점
- 1인 디자이너·개발자에게 가장 효율적인 스택
- 이터레이션 속도가 압도적 (하루 수 회 수정·배포 가능)
- 랜딩 페이지는 6개월간 20~50회 A/B 테스트 필요한데 이 구조가 최적
- 디두 본업 퀄리티와 일관성 유지

### 유의점
- 완벽한 1회 런칭에 집착하지 말고 2주 내 러프 런칭 후 데이터 기반 개선
- 카피가 디자인보다 전환율에 중요 — 카피 확정 후 Figma 작업
- 결제는 초기 MVP에서 수동(계좌이체·카드 링크)으로 시작, 가입 10곳 이후 정기결제 시스템 별도 프로젝트로 구축

## 10. 솔직한 최종 조언

이 아이템은 **"지금 아니면 안 되는 타이밍 × 정확한 타겟 × 적절한 가격"**의 삼박자가 맞아떨어진 케이스입니다. 1인 SaaS로서의 성공 조건을 다음과 같이 요약합니다.

### 반드시 지켜야 할 것

1. **자동화가 운명을 결정합니다** — 수동 운영은 200곳 이전에 반드시 한계 도달. 업체 등록부터 리포트까지 무인 파이프라인 구축이 최우선 과제

2. **디두 본업과의 유기적 연결 유지** — AI Place 단독 성공보다 "디두의 세일즈 깔때기"로 설계할 때 가장 큰 가치 창출. 두 사업의 브랜드 일관성이 중요

3. **신뢰도 이슈 제로 톨러런스** — 통계 수치 불일치, 템플릿 버그 같은 작은 결함이 저가 SaaS의 이탈률을 2배로 만듦. 완벽주의는 필요 없지만 정합성은 필수

4. **증명 도구를 제품의 핵심으로** — AI 검색 리포트가 이 서비스의 진짜 차별화 요소. 단순 리포트가 아닌 "우리 가게가 AI에 나왔다"는 증거를 매월 손에 쥐여주는 경험 설계

5. **법적 리스크 선제 대응** — 의료광고·리뷰 저작권은 매출이 커지면 커질수록 치명적. 베타 단계에서 법무 자문 1회는 투자할 가치 있음

### 하지 말아야 할 것

1. 전 업종·전 지역을 한 번에 펼치기 — 피부과 1개 지역부터 완성도 있게
2. 랜딩을 두 달씩 다듬기 — 2주 내 러프 런칭, 6개월간 점진 개선
3. 처음부터 정기결제 시스템 구축 — 수동 결제로 시작
4. "AI Place를 거대한 플랫폼으로" 목표 — 디두의 무기로 쓸 때 훨씬 성공 확률 높음
5. 완벽한 디자인 시스템 먼저 구축 — 랜딩 그리면서 시스템 확장하는 방식이 현실적

### 성공 가능성에 대한 최종 판단

독립 플랫폼으로만 본다면 성공 확률 **70%**, 디두의 부가 서비스이자 세일즈 무기로 통합 운영할 경우 **90%**로 판단합니다. 본인이 가진 자원(디두 브랜드·고객 기반·디자인 역량·Claude Code 활용력)과 이 아이템의 궁합은 업계에서 보기 드물 정도로 좋습니다.

**AI Place는 단순한 프로젝트가 아니라 디두의 2막을 열 수 있는 사업 아이템**으로 판단됩니다. 다만 그 가능성이 현실화되는 지점은 "얼마나 빨리 최소 기능으로 런칭하고, 실 사용자 데이터를 기반으로 반복 개선하느냐"에 달려 있습니다. 완성도보다 속도가 승부를 가릅니다.
